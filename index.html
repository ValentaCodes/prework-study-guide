HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • "!Doctype is a declaration that tells the browser what document type it should expect.
  • "Metadata" is data about our webpage that the browser uses to serve our webpage to users.
  • The style element defines CSS styles for the webpage.
  • The base element defines base URL for the webpage.
  • The link element connects external resources to HTML document.
  • The meta elementdefines metadata, charset, author, keywords, and viewpoint.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS Box Model: Every HTML element is essentially a box. Each box consists of margins outside elements, and the elements borders, padding, and content.
  • "*" applies a rule to all elements visible on page.
  • Text-align: is a positioning property.
  • The "rel" attribute: specifies relationship between current document and the linked document resouce.
  • The "href" attribute: specifies the URL of external resource.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b "branch name": creates a new branch and switches to it.
  • git add: adds modifications in the current working branch.
  • git -A: indicates that we want to add all changes.
  • git pull: receives the branches modifications into the local enviroment.
  • git push: pushes changes we have locally to our remote Github branch.
  • The "origin" keyword: indicates the source of the pull will be in the Github repo.
  • The "Main" keyword indicates the main branch.

  • Variable: Is a named container that allows us to store data in out code.
  • Control flow: Is the order in which a computer executes code in a script.

  • For Loops: Allows a single block of code to be executed over and over.
      • + Three important statements make up a for loop.
      1. Starting point.
      2. Condition.
      3. Iteration.
  • Functions: Creates a set of instructions that tells the computer how to perform a task.
      • + Different from "for" loops and conditional statements because functions don't automatically execute when a javascript file is run.
